implement when you have time and context"). The 11 points from that note were grouped into

`feature/todo-batch` behind a full-suite gate (pytest + ruff + mypy, all green).

## Integration notes

Merge order: `todo/group-a` → `todo/group-b` → `todo/group-c` (each `--no-ff`).

Conflicts resolved (all by keeping BOTH features working):
- `app/agent/session.py` `derive_title` docstring (a vs b) — merged into one docstring; the
  body already merged cleanly and skips BOTH `synthetic: True` (probe-leak, #6) and
  bracket-tagged (catalog injection, #11) messages.
- `ui/styles.css` — purely additive (group-a resource dashboard pane + responsive media
  query, group-b results-card styles); kept both blocks.
- `knowledge/conversation_style.md` — purely additive (group-b "Finding the right help /
  search_knowledge", group-c "After a benchmark — what to offer next"); kept both sections.

One integration fix beyond the conflicts: `app/main.py` `_history_items` originally (on
group-a) skipped only `synthetic: True` user messages. Group-b's once-per-session catalog
injection is bracket-tagged but NOT synthetic-flagged, so on resume it would have rendered as
a user bubble (a latent bug on group-b's own branch). `_history_items` now also skips
bracket-tagged (`content.startswith("[")`) injected user messages, mirroring `derive_title`,
so the integrated result is strictly cleaner than either branch alone.

Gate after all merges: full pytest suite GREEN (the 16 live evals in
`tests/flows/test_flows_live.py` stay skipif-gated and skipped), ruff clean, mypy clean.
