## API & feature reference points

The concrete API surface тАФ flags, env vars, CRD kinds, report field paths, and metric names тАФ that the agent should be able to COVER (offer/explain) or USE (drive/parse) when deploying llm-d stacks and running benchmarks. Distilled from every doc's `reference_points` and deduplicated.

### Benchmark CLI & lifecycle (`llmdbenchmark`)
- Subcommands: `plan`, `standup`, `smoketest`, `run`, `teardown`, `experiment`, `results`.
- Global: `--spec`/`-s` (e.g. `gpu`, `cicd/kind`, `guides/optimized-baseline`, `guides/multi-model-wva`, `pd-disaggregation`), `--namespace`/`-p`, `--dry-run`/`-n`, `--debug`/`-d`, `--skip`/`-z`, `--workspace`, `--steps` step-spec (`"0,3-5,9"` or `-s 10`).
- standup: `--methods`/`-t` (`standalone`|`modelservice`|`fma`|`kustomize`), `--models`/`-m`, `--monitoring`/`-f` and `--no-monitoring` (tri-state), `--gateway-class`, `--wva`/`-u`, `--skip-smoketest`, `--non-admin`/`-i`. Steps 00-10 (EnsureInfra, AdminPrerequisites, WorkloadMonitoring, ModelNamespace, HarnessNamespace, Deploy(Standalone/FMA/Kustomize), DeploySetup, DeployGaie, DeployModelservice).
- run: `--harness`/`-l`, `--workload`/`-w`, `--experiments`/`-e`, `--overrides`/`-o` (`key=value`, e.g. `concurrency=32,duration=300,max_tokens=512`), `--parallelism`/`-j`, `--endpoint-url`/`-U` (run-only mode), `--wait-timeout`, `--generate-config`/`-c`, `--analyze`, `--stack`, `--list-endpoints`, `--output`/`-r`, `--dataset`. Steps 00-11 (detect_endpoint, verify_model, render_profiles ... collect_results, upload_results, AnalyzeResultsStep step 11).
- smoketest: `-p`/`[-s 0|1|2]` тЖТ step 00 health_check (`/health`, `/v1/models`), step 01 inference_test (`/v1/completions` тЖТ `/v1/chat/completions`), step 02 validate_config; per-scenario validators (pd-disaggregation, precise-prefix-cache-aware, inference-scheduling, tiered-prefix-cache, wide-ep-lws, simulated-accelerators, cpu-example-ms, gpu-example, spyre-example).
- teardown: `--deep`/`-d` (delete ALL incl. PVC), `--stack` (partial). Steps 00-04 (Preflight, UninstallHelm, CleanHarness, DeleteResources, CleanClusterRoles); resource types deployment/httproute/inferencepool/gateway/configmap/job/pvc; VariantAutoscaling/HPA always deleted.
- experiment (DOE): `--experiments`/`-e`, `--parallel`/`-j N`, `--stop-on-error`, `--skip-teardown`; YAML `experiment:`/`setup:`/`treatments:` (or `run:`), `setup.constants`, dotted keys (e.g. `decode.parallelism.tensor`), `design.type` `full_factorial`/`proportional`, `total_matrix = setup x run`, `NA` placeholder, `experiment-summary.yaml`.
- results store: `results init` / `remote add|ls|rm` / `add <workspace>` / `push|pull <store>` / `status` / `ls prod -m 'llama-*'`; `.result_store/`, scenario/model/hardware/run-uid taxonomy.
- Harnesses: `inference-perf`, `guidellm`, `vllm-benchmark`, `inferencemax`, `nop`.
- guidellm CLI (interactive pod): `guidellm benchmark --target --rate-type sweep --max-seconds --model --data prompt_tokens=256,output_tokens=128`.
- existing-stack path: `run_only.sh -c config.yaml -o ./results`; `llmdbench-harness-launcher` / `llmdbench-${HARNESS}-launcher` pod; `workload-pvc` / `BENCHMARK_PVC` (RWX 200Gi); `envsubst < ${BENCHMARK_TEMPLATE} > config.yaml`.
- Telemetry: `--telemetry-enabled`/`--telemetry-provider=http`/`--telemetry-endpoint`; env `LLMDBENCH_TELEMETRY_ENABLED/_ENDPOINT/_API_KEY/_TOKEN/_AUTH_PROVIDER`.
- Tooling prereqs: REQUIRED `kubectl,helm,helmfile,jq,yq`; OPTIONAL `oc,kustomize,skopeo,rsync,make`; Python `>=3.11`/`>=3.12`.
